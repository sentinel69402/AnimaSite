Examples

Real use cases.

Focused scripts for specific scenarios, matching the examples folder included in the module.

01

Basic playback

The simplest way to play an animation. Create an Anima instance and call :PlayAnimation.

local Anima = require(game.ReplicatedStorage.Anima)

local anima = Anima.new(
    game.ReplicatedStorage.anims,
    game.Players.LocalPlayer
)

anima:PlayAnimation("Idle", {
    loop     = true,
    fadeTime = 0.2
})

02

Disabling Roblox Animate

Take full control of character animation by disabling the default Animate script. Just pass true as the fourth argument.

local anima = Anima.new(
    game.ReplicatedStorage.anims,
    game.Players.LocalPlayer,
    false,
    true
)

03

State machines

Manage high-level character states like Jump, Attack, or Idle without mixing that logic into your movement code.

The state machine lets you define transitions based on a shared context. Each state can have onEnter and onExit callbacks. Call :Update() every frame to evaluate transitions.

The state machine decides what the character is doing. Blend controllers handle how that motion looks.

local SM = anima:CreateStateMachine({
    initial = "Locomotion",
    context = { IsJumping = false }
})

SM:AddState("Jump", {
    onEnter = function()
        anima:PlayAnimation("Jump")
    end,
    transitions = {
        Locomotion = function(ctx)
            return not ctx.IsJumping
        end
    }
})

04

Locomotion blending

Keep Idle and Run tracks alive and smoothly shift weight based on speed. No snapping between states.

anima:CreateBlendController("Locomotion", {
    nodes = {
        { name = "Idle", min = 0,  max = 1 },
        { name = "Run",  min = 6,  max = 16 }
    }
})

local speed = humanoid.MoveDirection.Magnitude * humanoid.WalkSpeed
anima:UpdateBlend("Locomotion", speed)

05

Composition: states + blending

The real power of Anima: combine a state machine with a blend controller. The state machine handles intent, and the blend controller handles smooth motion.

local SM = anima:CreateStateMachine({
    initial = "Locomotion",
    context = { IsJumping = false }
})

SM:AddState("Locomotion", {
    transitions = {
        Jump = function(ctx) return ctx.IsJumping end
    }
})

SM:AddState("Jump", {
    onEnter = function() anima:PlayAnimation("Jump") end,
    onExit  = function() anima:StopAnimation("Jump", 0.1) end,
    transitions = {
        Locomotion = function(ctx) return not ctx.IsJumping end
    }
})

anima:CreateBlendController("Locomotion", {
    nodes = {
        { name = "Idle", min = 0, max = 1 },
        { name = "Run",  min = 0, max = 16 }
    }
})

RunService.Heartbeat:Connect(function()
    SM.Context.IsJumping = humanoid:GetState()
        == Enum.HumanoidStateType.Jumping

    if SM:GetState() == "Locomotion" then
        local speed = humanoid.MoveDirection.Magnitude
            * humanoid.WalkSpeed
        anima:UpdateBlend("Locomotion", speed)
    else
        anima:UpdateBlend("Locomotion", 0)
    end

    SM:Update()
end)

06

Overlay animations

Layer actions like punching or casting on top of movement using animation priorities.

anima:PlayAnimation("Punch", {
    priority   = Enum.AnimationPriority.Action,
    stopOthers = false,
    fadeTime   = 0.1
})

07

NPC & custom rig support

Anima works with non-player characters and AnimationController rigs. Just pass the model instead of a player.

local npcAnima = Anima.new(
    game.ReplicatedStorage.npcAnims,
    workspace.NPC
)

npcAnima:PlayAnimation("Patrol", { loop = true })

08

Cleanup

When you're done with an instance, like when a character dies or an NPC despawns, call :Destroy() to clean up tracks and connections.

anima:Destroy()
