About

Why Anima exists.

Roblox animation code turns into a mess. Anima was made to fix that without trying to own your gameplay.

The mess

Standard Roblox animation code often turns into...

Duplicated track loading

The same load-and-play boilerplate copy-pasted across scripts, every time a new animation is needed.

Priority conflicts

Your script fights the default Animate script. Both try to play animations at the same time, and one wins unpredictably.

Jarring transitions

Stop-and-start playback causes animations to pop and snap instead of blending smoothly.

Spaghetti movement checks

Hard-to-read speed, jump, and state checks scattered through gameplay scripts with no clear structure.

Anima's approach

Friendly, not dogmatic.

Anima doesn't pretend to be a full controller or a physics engine. It focuses on one thing: animation playback. And it does it well.

Tracks cached once

AnimationTracks are loaded once and stored. No runtime hitches from loading assets when the character starts moving.

Real weight blending

Animations stay alive at weight 0 during blends. It uses actual interpolation instead of just stopping and starting tracks.

Composable systems

High-level features like state machines and blend controllers are completely optional. Use what you need.

Boundaries

What Anima isn't.

To keep things lightweight, this library specifically avoids:

Movement logic or physics

Anima doesn't move your character. That's your game's job.

Input handling

No key bindings, no controller support. Anima responds to your commands, not player input directly.

Character controllers

No custom character controller. Use Anima alongside your own or Roblox's built-in systems.

Gameplay decisions

Anima handles how animations look and transition, but it never decides how your game works.

Built for real work

Designed for actual Roblox projects.

Many Roblox developers end up with one script handling movement, another handling animation, and a third fighting Roblox's default Animate script. Anima keeps things clear, composable, and easy to maintain. Point it at a folder or a table of IDs, tell it what to play, and it handles the rest.

Anima is designed to be predictable and composable. It shouldn't have side effects that surprise you. If it does, that's a bug.

Feature summary

The heart of the module.

  • Zero-cost caching load animation assets once, reuse them forever.
  • 1D blend controllers smooth transitions driven by a single value.
  • Optional state machine separate intent from motion cleanly.
  • Folder-based setup point at a folder, no manual config.
  • NPC & rig support works with Humanoids and AnimationControllers.
  • Singleton pattern one instance per player, preventing duplicated logic.
  • Animate conflict handling disable Roblox's Animate script with a flag.
  • Well documented clear internal comments and a clean API.
