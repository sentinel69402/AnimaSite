API Reference

How to use Anima.

Everything you need to know, from creating an instance to managing blend controllers and state machines.

Types

Core type definitions.

type PlaybackConfig = {
    fadeTime:    number?,
    weight:      number?,
    priority:    Enum.Priority?,
    loop:        boolean?,
    stopOthers:  boolean?
}

type BlendNode = {
    name: string,
    min:  number,
    max:  number
}

type State = {
    onEnter:     (() -> ())?,
    onExit:      (() -> ())?,
    transitions: { [string]: (context: table) -> boolean }?
}

Initialization

Creating an Anima instance.

Anima.new(source, subject, debug?, disableAnimate?)

source can be a Folder or a Table ({[string]: number | string}). subject can be a Player or a Model (NPC/custom rig). Automatically detects a Humanoid or AnimationController. Returns a singleton when a Player is provided.

:LoadAnimations()

Manually re-scan the animation folder and load any new entries.

:Cache()

Reloads character animator and tracks. Call this on respawn to rebind everything.

:WatchForChanges()

Enables live-reloading of animations during development. Useful for iteration in Studio.

Playback

Playing and stopping animations.

:PlayAnimation(name, config?) → AnimationTrack?

Start a named animation. Pass a PlaybackConfig to control fading, looping, weight, and priority.

:StopAnimation(name, fadeTime?) → boolean

Fade out a specific animation. Returns whether the track was found and stopped.

:PauseAnimation(name) / :ResumeAnimation(name)

Pause a playing track and resume it later without restarting.

:PlayLooped(name, loopCount?)

Play an animation a specific number of loops, then stop automatically.

:SetAnimation(name, id)

Dynamically set or replace an animation using an ID or Instance.

:SetAnimationSpeed(name, speed)

Persistently sets the playback speed for an animation.

:SetWeight(name, weight, fadeTime?)

Manually set track weight for layering and blending control.

:FadeAllOut(fadeTime?)

Fade out every currently playing animation.

Queries

Asking Anima about state.

:GetAnimationTrack(name) → AnimationTrack?

Returns the raw AnimationTrack for a named animation, if loaded.

:GetPlayingTracks() → { string }

Returns a list of animation names that are currently playing.

:GetAnimationProgress(name) → (progress, isPlaying)

Get the playback progress (0 to 1) and whether the animation is currently active.

:IsAnimationPlaying(name) / :IsAnimationLoaded(name)

Quick boolean checks for animation state.

Grouping & Sequencing

Batch operations and queues.

:QueueAnimations(names, config?)

Play a list of animations in sequence, one after the other.

:playSequence(names, fadeTime)

Simplified sequential playback with a shared fade time.

:setAnimationTag(name, tag) / :fadeOutTag(tag, fadeTime?)

Tag animations for batch control. Fade out an entire group at once.

Systems

Blend controllers & state machines.

:CreateBlendController(name, profile) → BlendController

Set up smooth 1D blending, typically for locomotion. Requires a list of BlendNode entries.

:UpdateBlend(name, value)

Feed a numeric value (e.g. speed) to a blend controller and let Anima mix the animation weights.

:CreateStateMachine(config) → StateMachine

Create a state machine for managing high-level character intent. Provide an initial state and context table.

BlendController methods

:Update(value)

Manually update the blend tree weights based on the given value.

:Destroy()

Clean up the controller and release resources.

StateMachine methods

:AddState(name, state)

Register a new state with optional onEnter, onExit, and transitions.

:GetState() → string

Returns the current state name.

:SetState(name)

Force a transition to a specific state.

:Update()

Evaluate transitions based on the current context. Call this every frame.

:Lock(duration)

Prevent transitions for a set amount of time. Useful for uninterruptible animations.

.Context

The shared table for transition data. Update this before calling :Update().

Lifecycle

Events and cleanup.

:getPlaySignal() / :getStopSignal()

Returns custom Signal objects. Listeners receive: (animationName, player?, character).

:setAnimationCallbacks(name, callbacks)

Hook into animation state changes or markers for custom reactions.

:Destroy()

Clean up all tracks and event connections. Call when you're done with the instance.
