Animations that don't fight back.

Anima is a lightweight Roblox animation library that handles caching, blending, and state transitions so you can focus on making your game feel good.

Explore the API See examples

Zero-cost caching

Tracks are loaded once and stored. No runtime hitches from loading assets on the fly, just instant playback.

Real blending

Animations stay alive at weight 0 during transitions. No popping, no jarring cuts, just smooth interpolation.

NPC & custom rigs

Works out of the box for players, NPCs, or any model with a Humanoid or AnimationController.

The problem

Roblox animation code gets messy fast.

Standard approaches turn into a web of duplicated track loading, scripts fighting over priorities, and hard-to-read movement checks scattered everywhere. Anima takes a different approach.

You decide what should play. Anima handles how it behaves.

  • Folder-based setup point it at a folder and it finds everything. No manual config.
  • Singleton pattern one instance per player, no duplicated logic.
  • Animate conflict handling disable Roblox's default Animate script with one flag.
  • Optional state machine keep intent separate from motion without mixing things up.

Core systems

Blend controllers & state machines.

High-level tools that stay out of your way until you need them.

1D Blend Controllers

Smoothly interpolate between Idle, Walk, and Run based on a single value, usually speed. No snapping.

State Machine

Define high-level intent like "jumping" or "attacking" without mixing it into your physics or input code.

Composable

Use states and blending together. Let the state machine decide what, and the blend controller decide how.

Quick start

Up and running in seconds.

local Anima = require(game.ReplicatedStorage.Anima)

local anima = Anima.new(
    game.ReplicatedStorage.anims,
    game.Players.LocalPlayer,
    false,
    true
)

anima:PlayAnimation("Idle", {
    loop     = true,
    fadeTime = 0.2
})

Philosophy

Predictable and composable.

Anima handles how animations look and transition, but it doesn't decide how your game works. It specifically avoids movement logic, input handling, character controllers, and gameplay decisions. If it has side effects that surprise you, that's a bug.
